# 开发流程

## 1. 需求接收与范围确认

1. 明确目标与预期结果。
2. 确认受影响页面/模块。
3. 在任务描述中记录范围。

## 2. 设计与规划

1. 必要时更新需求或架构文档。
2. 确认用户流程与验收标准。
3. 将工作拆分为可评审的小任务。

## 3. 开发实现

1. 从 `main` 拉出分支。
2. 使用小步提交实现变更。
3. 除非范围要求调整，否则保持遗留行为稳定。

## 4. 验证

1. 在桌面端和移动端完成功能检查。
2. 检查 SEO 元数据与链接有效性。
3. 进行性能健康检查（Lighthouse）。

## 5. 评审与合并

1. 发起 PR，若 UI 有变更需附上下文与截图。
2. 完成检查清单（`docs/pr-checklist.md`）。
3. 通过评审且 CI 通过后再合并。

## 6. 发布与观察

1. 按标准流程执行部署。
2. 验证生产页面状态。
3. 监控告警与错误。
