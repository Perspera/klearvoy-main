# Next.js 迁移计划

## 目标

在尽量降低停机风险的前提下，将遗留 `index.htm` 静态页面迁移到可维护的 Next.js 架构。

## 迁移原则

- 尽可能保留对 SEO 关键的内容和 URL。
- 在功能对齐确认前，保持旧站可用。
- 分阶段迁移，并设置可衡量的检查节点。

## 迁移阶段

### 阶段 1：基础建设

- 创建新的 Next.js 仓库。
- 配置 Lint、格式化、CI/CD 与环境管理。
- 定义设计令牌与共享布局组件。

### 阶段 2：页面对齐

- 还原当前 `index.htm` 的首页核心分区。
- 将媒体资源迁移到受管静态目录或 CDN。
- 验证响应式表现与元数据完整性。

### 阶段 3：质量加固

- 增加 `robots.txt`、`sitemap.xml` 与 404 页面。
- 接入数据分析与错误监控。
- 完成性能调优和跨浏览器检查。

### 阶段 4：切换上线

- 部署预发布环境并完成对齐验证。
- 切换生产域名路由。
- 持续监控错误与可用性 48 小时。

## 退出标准

- 核心用户路径在移动端和桌面端均可用。
- SEO 元数据与索引文件有效。
- 可用性与错误监控已启用。
- 回滚路径已验证。
